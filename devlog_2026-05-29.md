# Dev Log — 2026-05-29

## Summary

Major session focused on building out the cumulative dealer delta flow system, options flow monitoring improvements, UI overhaul, and stability fixes.

---

## Features Built

### 1. Cumulative Dealer Delta — Full-Book Chain Diff
Previously the delta chart only accumulated from streaming alerts (top 40 strikes). Changed to diff the entire SPX option chain every 60 seconds via REST API — every strike, every expiry up to 60 DTE. Direction inferred from last price vs bid/ask midpoint.

### 2. 0DTE-Only Delta Chart
Added a second cumulative delta chart tracking only contracts expiring today. The all-expiry chart shows the full dealer hedging picture; the 0DTE chart isolates same-day gamma which directly moves ES futures in real time. Both charts persist across restarts.

### 3. Flow Alerts Persistence
Flow alerts now save to `data/flow_alerts_YYYY-MM-DD.json` after every alert. On restart the feed is restored from disk. Each day gets its own file for historical reference.

### 4. Flow Alert Markers on SPX Chart
Bullish flow alerts (PUT SELL, CALL BUY) render as green up arrows on the SPX price chart. Bearish flow alerts (PUT BUY, CALL SELL) render as red down arrows. Uses LightweightCharts v5 `createSeriesMarkers` API (v5 removed `setMarkers` from the series directly). Markers persist across page reloads via the flow alert cache.

### 5. 0DTE Streaming — Full Strike Coverage
Changed `get_watch_contracts()` in gex.py: 0DTE now streams ALL strikes within ±5% of spot (was top-40 OI only). Multi-expiry keeps the top-40 OI filter. Result: ~256 0DTE contracts watched vs ~80 before.

### 6. Flow Feed Redesign
- Full-width section (no longer sharing grid with delta chart)
- Green background for bullish alerts (PUT SELL, CALL BUY), red for bearish
- Timestamp + live "Xm Ys ago" counter ticking every second
- Total premium spent shown (contracts × last price × $100 multiplier)
- Side and direction labels colored by price impact, not option type

### 7. Path Fixes
`data/` was resolving to `src/data/` when running from the `src/` directory. Fixed `gex.py`, `delta_flow.py`, and `price_history.py` to resolve `data/` relative to project root using `__file__`.

### 8. ES Candle Cache Cap
ES CSV has 700K+ rows. Now caps in-memory cache to 5 days on startup. Full CSV preserved on disk. Prevents silent crash during startup.

### 9. Chart Times — Local Timezone
All LightweightCharts time axes, crosshairs, and tick marks switched from hardcoded `America/New_York` to browser local time.

### 10. SPY/QQQ Commented Out
Removed SPY and QQQ from GEX refresh loop and frontend to reduce API load. SPX-only now. Each 60s cycle is one chain fetch instead of three.

### 11. Watch List Logging
Every 60s GEX refresh logs the exact contract count, 0DTE vs MULTI split, and strike range being streamed.

## Bug Fixes

### numpy 2.4.6 Crash (Python 3.12 segfault)
App was crashing with `0xc0000005` access violation in `python312.dll` every 2-3 minutes. Root cause: numpy 2.x has heap corruption bugs under multi-threaded Python 3.12 on Windows. Fix: downgrade to `numpy==1.26.4`.

### File Truncation
Multiple files were silently truncated during edits (Edit tool byte limits on large files). Developed a pattern of running AST syntax checks after every edit session and using Python scripts to repair truncated lines.

### LWC v5 Markers API
`setMarkers` was removed from the series API in LightweightCharts v5. Replaced with `LWC.createSeriesMarkers(candleSeries, [])` which returns a primitive with its own `setMarkers`. Both session markers and flow markers now share one primitive to avoid overwriting each other.

### SPX Chain Overflow
`strikeCount=300` exceeded Schwab's response body limit. Tested 175 (overflow), settled on 150 which loads cleanly. At $5 strike spacing: ±$375 from spot, covers ~7,200–7,950 at current SPX levels.

### git index.lock / HEAD.lock
Git lock files left from crashed processes prevented commits. Cleared manually with `del .git\index.lock` and `del .git\HEAD.lock`.
